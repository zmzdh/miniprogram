## 1. 协议概述

### 1.1 设计目标

- **保持一主多从**：保留 Master/Slave/Monitor/Sniffer 四种工作模式，16 通道从机
- **更强的完整性**：改用标准 CRC-8-ATM (0x07) 校验
- **双向确认**：命令-应答机制，携带状态码
- **设备发现**：PING/PONG 发现设备并获取能力信息
- **统一帧格式**：所有命令使用相同的 16 字节帧结构
- **向后兼容考虑**：帧长度保持 16 字节，便于在相同硬件上实现

### 1.2 工作模式

| 模式 | 值 | 说明 |
|------|-----|------|
| MONITOR | 0x00 | 监听模式：仅监听 LIN 总线，不参与通信 |
| MASTER | 0x01 | 主模式：作为主节点发送帧头和数据 |
| SLAVE | 0x02 | 从模式：作为从节点响应主节点请求（16通道） |
| SNIFFER | 0x03 | 嗅探模式：捕获所有总线流量（含 break） |

模式切换需间隔 ≥100ms。

| 偏移 | 长度 | 值 | 说明 |
|------|------|-----|------|

## 3. 帧格式

### 3.1 通用帧结构 (16 字节)

所有命令遵循统一的 16 字节帧结构：

| 偏移 | 字段 | 大小 | 说明 |
|------|------|------|------|
| 0 | CMD | 1 | **命令码** (0x01-0x7F) |
| 1 | CHANNEL | 1 | **通道号** (0-15)，非从机命令填 0 |
| 2 | ID | 1 | **LIN ID** (0x00-0x3F) 或子命令参数 |
| 3 | FLAGS | 1 | **标志位**，详见 §3.1.1 |
| 4 | LEN | 1 | **数据长度** (0-8) |
| 5-12 | DATA[0..7] | 8 | **数据负载** |
| 13 | LIN_CKS | 1 | **LIN 校验和** （非 LIN 命令填 0x00） |
| 14 | STATUS | 1 | **状态码** — 请求填 0x00，应答返回状态 |
| 15 | CRC8 | 1 | **CRC-8 帧校验** |

#### 3.1.1 FLAGS 标志位定义

| 位 | 名称 | 说明 |
|----|------|------|
| bit0 | DIR | **传输方向**：0=发送（主机→从机）, 1=接收（从机→主机） |
| bit1-2 | CHECK_TYPE | **校验类型**：0=无校验, 1=经典(V1.x), 2=增强(V2.x), 3=保留 |
| bit3 | ACK_REQ | **应答请求**：1=要求接收方回复确认 |
| bit4 | EXT | **扩展数据**：1=数据区包含扩展头（预留） |
| bit5-7 | RSV | 保留位（填 0） |

**与 LINTest-M 的区别**：原来 DIR 和 CHECK_TYPE 各占一个完整字节，现合并到 FLAGS 中，节省出一个字节作为 STATUS。

### 3.2 CRC-8 计算

采用标准 **CRC-8-ATM (CRC-8-Dallas)** 算法：

- 多项式: **0x07** (x^8 + x^2 + x + 1)
- 初始值: **0x00**
- 结果异或: **0x00**
- 校验范围: 前 15 字节 (offset 0-14)

```
CRC8 = crc8_atm(frame[0..14])    // 多项式 0x07, 初始 0x00
```

参考实现（JavaScript）：

```javascript
function calcCRC8(bytes) {
  let crc = 0x00
  for (const b of bytes) {
    crc ^= b
    for (let i = 0; i < 8; i++) {
      crc = (crc & 0x80) ? ((crc << 1) ^ 0x07) : (crc << 1)
      crc &= 0xFF
    }
  }
  return crc
}
```

**与 LINTest-M 的区别**：原协议使用 `sum(bytes[0..14])` 的补码 +1，抗干扰能力弱。CRC-8-ATM 可检测所有单/双比特错误和大部分突发错误。

### 3.3 LIN 校验和

保持与 LIN 标准一致的校验和计算，兼容所有三种类型：

| CHECK_TYPE | 算法 | 适用范围 |
|-----------|------|---------|
| 0 (NONE) | 返回 0x00 | 调试/非标 |
| 1 (CLASSIC) | `~(data[0] + ... + data[n-1])` | LIN V1.x, V1.3 |
| 2 (ENHANCED) | `~(PID + data[0] + ... + data[n-1])` | LIN V2.x, V2.1 (默认) |

数据长度 LEN 为实际有效数据字节数（1-8）。

## 5. 状态码

STATUS 字节和 ERROR 命令中的 DATA[0] 使用统一的状态码定义：

| 状态码 | 名称 | 说明 |
|--------|------|------|
| 0x00 | SUCCESS | 成功 |
| 0x01 | ERR_UNKNOWN_CMD | 未知命令码 |
| 0x02 | ERR_INVALID_PARAM | 参数无效 |
| 0x03 | ERR_CRC | 帧 CRC 校验失败 |
| 0x04 | ERR_MODE | 当前模式不支持该操作 |
| 0x05 | ERR_TIMEOUT | LIN 总线超时（无从机响应） |
| 0x06 | ERR_CHANNEL | 通道号无效或未使能 |
| 0x07 | ERR_LIN_ID | LIN ID 无效或保留 |
| 0x08 | ERR_BUS | LIN 总线错误 |
| 0x09 | ERR_DATA_LEN | 数据长度超出范围 |
| 0x0A | ERR_CHECKSUM | LIN 校验和不匹配 |
| 0x0B | ERR_BUSY | 设备忙，请稍后重试 |
| 0x10-0x2F | 厂商自定义 | 各厂商自行定义 |
| 0x30-0xFF | 保留 | — |

## 6. 时序与流程

### 6.1 设备发现流程

```
手机                          LINHelper 设备
  │                               │
  │────── PING (0x01) ──────────→│
  │                               │
  │←───── PONG (0x02) ──────────│   (含能力信息)
  │                               │
  │────── SET_MODE (0x11) ──────→│   (配置模式 + 波特率)
  │                               │
  │←───── MODE_ACK (0x12) ──────│   (确认模式切换)
  │                               │
```

### 6.2 主模式发送

```
手机 (MASTER)                  LINHelper 设备                    LIN 从节点
  │                               │                               │
  │── MASTER_SEND (0x21) ───────→│                               │
  │                               │── LIN Break ────────────────→│
  │                               │── LIN Sync (0x55) ──────────→│
  │                               │── LIN PID ──────────────────→│
  │                               │── LIN Data[0..n-1] ─────────→│
  │                               │── LIN Checksum ─────────────→│
  │                               │                               │
  │←──── STATUS=0x00 ───────────│  (发送完成)                     │
  │                               │                               │
```

若 ACK_REQ=1，设备在确认 LIN 帧发送到总线后回复 ACK。

### 6.3 读取从节点数据

```
手机 (MASTER)                  LINHelper 设备                    LIN 从节点
  │                               │                               │
  │── READ_SLAVE (0x22) ────────→│                               │
  │                               │── LIN Break ────────────────→│
  │                               │── LIN Sync (0x55) ──────────→│
  │                               │── LIN PID (with P1,P0) ─────→│
  │                               │←── LIN Data[0..n-1] ────────│
  │                               │←── LIN Checksum ────────────│
  │                               │                               │
  │←── SLAVE_DATA (0x23) ──────│  (返回从节点数据)               │
  │                               │                               │
```

### 6.4 从机模式

```
手机                             LINHelper 设备 (SLAVE)           LIN 主节点
  │                               │                               │
```

### 7.2 CRC-8-ATM 参考实现

**C 语言**：
```c
uint8_t calc_crc8(const uint8_t* data, uint8_t len) {
    uint8_t crc = 0x00;
    for (uint8_t i = 0; i < len; i++) {
        crc ^= data[i];
        for (uint8_t j = 0; j < 8; j++) {
            if (crc & 0x80) {
                crc = (crc << 1) ^ 0x07;
            } else {
                crc <<= 1;
            }
        }
    }
    return crc;
}
```

**Python**：
```python
def calc_crc8(data: bytes) -> int:
    crc = 0
    for b in data:
        crc ^= b
        for _ in range(8):
            if crc & 0x80:
                crc = ((crc << 1) ^ 0x07) & 0xFF
            else:
                crc = (crc << 1) & 0xFF
    return crc
```

### 7.3 LIN 校验和参考实现

```javascript
// 计算 Protected ID
function buildProtectedId(id) {
  const b0 = (id >> 0) & 0x01
  const b1 = (id >> 1) & 0x01
  const b2 = (id >> 2) & 0x01
  const b3 = (id >> 3) & 0x01
  const b4 = (id >> 4) & 0x01
  const b5 = (id >> 5) & 0x01
  const p0 = b0 ^ b1 ^ b2 ^ b4
  const p1 = (b1 ^ b3 ^ b4 ^ b5) ^ 0x01
  return (id | (p0 << 6) | (p1 << 7)) & 0xFF
}

// 经典校验 (LIN V1.x)
function classicChecksum(data) {
  let sum = data.reduce((a, b) => a + b, 0)
  while (sum > 0xFF) sum = (sum & 0xFF) + (sum >> 8)
  return (~sum) & 0xFF
}

// 增强校验 (LIN V2.x)
function enhancedChecksum(linId, data) {
  const pid = buildProtectedId(linId)
  let sum = pid + data.reduce((a, b) => a + b, 0)
  while (sum > 0xFF) sum = (sum & 0xFF) + (sum >> 8)
  return (~sum) & 0xFF
}
```
